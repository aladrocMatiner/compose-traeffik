# Rocket.Chat service

<a id="overview"></a>
## Overview

Rocket.Chat is an optional real application module for this stack. It runs behind Traefik and includes MongoDB (replica set init helper) and NATS in the `rocketchat` profile.

<a id="location"></a>
## Where it lives

- `services/rocketchat/compose.yml`
- `services/rocketchat/config/rocketchat.env.example` (tracked fallback for compose parsing)
- `services/rocketchat/rendered/` (generated by `make rocketchat-bootstrap`, gitignored)

<a id="run"></a>
## How it runs

Bootstrap rendered config artifacts:
```bash
make rocketchat-bootstrap
```

Start the profile:
```bash
make rocketchat-up
```

Check status:
```bash
make rocketchat-status
```

View logs:
```bash
make rocketchat-logs
```

Stop the profile:
```bash
make rocketchat-down
```

<a id="configuration"></a>
## Configuration

Relevant env vars in `.env.example`:
- `ROCKETCHAT_VERSION`, `ROCKETCHAT_IMAGE`
- `ROCKETCHAT_HOSTNAME`, `ROCKETCHAT_PORT`
- `ROCKETCHAT_MONGODB_*`, `ROCKETCHAT_NATS_VERSION`
- `ROCKETCHAT_RENDERED_ENV_PATH`
- `ROCKETCHAT_OBSERVABILITY_ENABLED`, `ROCKETCHAT_METRICS_PORT`
- `ROCKETCHAT_KEYCLOAK_ENABLED`, `ROCKETCHAT_KEYCLOAK_*`
- `DEV_DOMAIN`, `TLS_CERT_RESOLVER`

Rendered artifacts (gitignored):
- Rocket.Chat runtime env: `services/rocketchat/rendered/rocketchat.env`
- Keycloak setup checklist: `services/rocketchat/rendered/keycloak-custom-oauth.md`

Keycloak integration:
- Optional and disabled by default.
- This repo renders a checklist/runbook (callback URL + endpoint paths) from `.env` values.
- Custom OAuth provider creation in Rocket.Chat remains a manual UI/admin task.

Observability hooks:
- Optional and disabled by default (`ROCKETCHAT_OBSERVABILITY_ENABLED=false`).
- When enabled, the rendered config turns on Rocket.Chat Prometheus settings and labels the container for scrape discovery.
- Metrics are not exposed publicly via Traefik by default.

<a id="ports"></a>
## Ports, networks, volumes

- Public endpoint: `https://rocketchat.${DEV_DOMAIN}` via Traefik (no direct host port published for the app)
- Internal service ports (container-only): Rocket.Chat `3000`, metrics `9458`, MongoDB `27017`, NATS `4222/8222`
- Networks: `proxy` and private `rocketchat-internal`
- Volumes: `rocketchat-uploads`, `rocketchat-mongodb-data`

<a id="security"></a>
## Security notes

- Add `rocketchat` to `ENDPOINTS` (or your DNS/hosts mapping) for local name resolution.
- Preflight validation fails if the rendered Rocket.Chat env file is missing; run `make rocketchat-bootstrap` first.
- Keycloak inputs are validated only when `ROCKETCHAT_KEYCLOAK_ENABLED=true` (HTTPS issuer and required credentials).
- Rocket.Chat is routed by Traefik, so it uses the same TLS mode selected for the stack (self-signed, LE, or optional step-ca).

<a id="troubleshooting"></a>
## Troubleshooting

- `make rocketchat-up` fails preflight about rendered config: run `make rocketchat-bootstrap` and verify `services/rocketchat/rendered/rocketchat.env` exists.
- Rocket.Chat waits on dependencies: inspect `make rocketchat-logs` for `rocketchat-mongodb`, `rocketchat-mongodb-init`, and `rocketchat-nats`.
- Keycloak login button not appearing: confirm manual Custom OAuth setup in Rocket.Chat admin UI and compare values with `services/rocketchat/rendered/keycloak-custom-oauth.md`.
- Local TLS trust problems (step-ca or self-signed): review the TLS mode guides and CA trust steps in the root docs.
