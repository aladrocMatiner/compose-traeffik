# wg-easy service (WireGuard)

<a id="overview"></a>
## Overview

`wg-easy` provides a WireGuard server with a web UI for managing peers and exporting client configurations.
In this project, the UI is exposed behind Traefik over HTTPS, while the WireGuard tunnel uses a host UDP port.

<a id="location"></a>
## Where it lives

- `services/wg-easy/compose.yml`
- `services/wg-easy/data/` (runtime state; gitignored)
- `scripts/wg-bootstrap.sh` (bootstrap admin `.env` variables)

<a id="run"></a>
## How it runs

Bootstrap admin variables in `.env` first:
```bash
make wg-bootstrap
```

Start the service:
```bash
make wg-up
```

Inspect status/logs:
```bash
make wg-status
make wg-logs
```

Stop the service:
```bash
make wg-down
```

<a id="configuration"></a>
## Configuration

Relevant env vars in `.env.example`:
- `WG_EASY_IMAGE`
- `WG_UI_HOSTNAME`
- `WG_UI_MIDDLEWARES`
- `WG_INSECURE` (must remain `false` in this integration)
- `WG_BIND_ADDRESS`
- `WG_ALLOW_NONLOCAL_BIND`
- `WG_SERVER_PORT`
- `WG_SERVER_ENDPOINT`
- `WG_INIT_ENABLED`
- `WG_INIT_USERNAME`
- `WG_INIT_PASSWORD` (generated by `make wg-bootstrap`)
- `TLS_CERT_RESOLVER`
- `DEV_DOMAIN`

Bootstrap/rotation notes:
- `make wg-bootstrap` fills missing `WG_INIT_*` values in `.env` and does not overwrite existing values by default.
- To rotate supported bootstrap values, use `WG_BOOTSTRAP_ARGS=--force make wg-bootstrap`.

<a id="ports"></a>
## Ports, networks, volumes

- Ports:
  - UDP `WG_BIND_ADDRESS:WG_SERVER_PORT -> 51820/udp` (WireGuard tunnel)
  - UI TCP port is **not** published to the host (served via Traefik)
- Networks:
  - `proxy`
- Volumes:
  - `services/wg-easy/data` -> `/etc/wireguard`

<a id="security"></a>
## Security notes

- The admin UI is routed through Traefik HTTPS and `WG_INSECURE=true` is blocked by preflight validation.
- Non-loopback UDP exposure requires explicit acknowledgement: `WG_ALLOW_NONLOCAL_BIND=true`.
- The container uses explicit capabilities/devices and does not use `privileged: true` by default.
- Runtime state under `services/wg-easy/data/` can contain sensitive material and is gitignored.

Host prerequisites (common Linux setups):
- `/dev/net/tun` available
- Docker engine with required capabilities/devices support
- Kernel support for WireGuard / networking forwarding behavior as required by your host

<a id="troubleshooting"></a>
## Troubleshooting

- **UI not reachable at `https://wg.<DEV_DOMAIN>`**
  - Check `make wg-status`, `make logs`, and Traefik logs.
  - Ensure `WG_UI_HOSTNAME` and `DEV_DOMAIN` match your hosts/DNS setup.
  - Add `wg` to `ENDPOINTS` and re-run `make hosts-apply` if using hosts mapping.

- **Preflight blocks WireGuard start**
  - Check `WG_BIND_ADDRESS`, `WG_ALLOW_NONLOCAL_BIND`, `WG_SERVER_PORT`, `WG_SERVER_ENDPOINT`, and `WG_INSECURE` in `.env`.

- **Container fails to start with TUN/capability errors**
  - Confirm `/dev/net/tun` exists and Docker can pass it through.
  - Review host/kernel support and container runtime restrictions.
