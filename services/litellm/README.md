# LiteLLM service

<a id="overview"></a>
## Overview

LiteLLM is an optional profile that provides an OpenAI-compatible LLM proxy/router behind Traefik.

<a id="location"></a>
## Where it lives

- `services/litellm/compose.yml`
- `services/litellm/config.yaml`

<a id="run"></a>
## How it runs

Bootstrap LiteLLM secrets in `.env`:
```bash
make litellm-bootstrap
```

Start the service:
```bash
make litellm-up
```

Standalone edge mode (Traefik + LiteLLM only):
```bash
make litellm-standalone-up
```

Verify health (pinned LiteLLM v1.81.x endpoint):
```bash
curl -sk "https://${LITELLM_HOSTNAME:-llm}.${DEV_DOMAIN}/health/liveliness" \
  -H "Authorization: Bearer ${LITELLM_MASTER_KEY}"
```

Verify the browser/docs/admin surface on the dedicated hostname (Traefik BasicAuth prompt expected):
```bash
curl -skI "https://${LITELLM_UI_HOSTNAME:-llm-admin}.${DEV_DOMAIN}/ui"
```

Example API request (works with the default local route if your local/LAN Ollama-compatible backend is available):
```bash
curl -sk "https://${LITELLM_HOSTNAME:-llm}.${DEV_DOMAIN}/chat/completions" \
  -H "Authorization: Bearer ${LITELLM_MASTER_KEY}" \
  -H 'Content-Type: application/json' \
  -d "{\"model\":\"${LITELLM_LOCAL_MODEL_ALIAS:-local-ollama}\",\"messages\":[{\"role\":\"user\",\"content\":\"ping\"}]}"
```

<a id="configuration"></a>
## Configuration

Relevant env vars in `.env.example`:
- `LITELLM_IMAGE`
- `LITELLM_HOSTNAME`
- `LITELLM_UI_HOSTNAME`
- `LITELLM_PORT`
- `LITELLM_MIDDLEWARES`
- `LITELLM_UI_MIDDLEWARES`
- `LITELLM_MASTER_KEY` (generated by `make litellm-bootstrap`)
- `LITELLM_SALT_KEY` (generated by `make litellm-bootstrap`)
- `LITELLM_UI_BASIC_AUTH_USER` / `LITELLM_UI_BASIC_AUTH_PASSWORD` (generated by `make litellm-bootstrap`)
- `LITELLM_UI_BASIC_AUTH_HTPASSWD_PATH` (htpasswd path generated/managed by `make litellm-bootstrap`)
- `LITELLM_LOCAL_API_BASE` (default local/LAN Ollama-compatible endpoint)
- `LITELLM_LOCAL_MODEL_ALIAS` / `LITELLM_LOCAL_MODEL_REF` (default local route alias/model)
- `OPENAI_API_KEY` (optional)
- `ANTHROPIC_API_KEY` (optional)
- `OPENROUTER_API_KEY` (optional)
- `DEV_DOMAIN`
- `TLS_CERT_RESOLVER`

Routing/model config is stored in `services/litellm/config.yaml` and should reference secrets via `os.environ/<VAR>`.
The config template includes a default local inference route, so you can point LiteLLM at another machine on your LAN by changing only `.env`.

<a id="ports"></a>
## Ports, networks, volumes

- Ports: container port `4000` (not published to host)
- Network: `proxy` (`traefik-proxy`)
- Volume: `services/litellm/config.yaml` -> `/app/config.yaml` (read-only)

<a id="security"></a>
## Security notes

- The service is exposed only via Traefik on `https://${LITELLM_HOSTNAME:-llm}.${DEV_DOMAIN}`.
- Browser/docs/admin routes are exposed on `https://${LITELLM_UI_HOSTNAME:-llm-admin}.${DEV_DOMAIN}` with Traefik BasicAuth.
- LiteLLM API auth is enabled by default via `LITELLM_MASTER_KEY`.
- UI BasicAuth credentials are separate from the LiteLLM API master key.
- Rotate LiteLLM secrets with `LITELLM_BOOTSTRAP_ARGS=--force make litellm-bootstrap` and update clients.
- The default local backend (`LITELLM_LOCAL_API_BASE`) is intended for trusted local/LAN use; secure that backend separately if it is remote.
- Provider API keys remain optional in `.env`; requests fail until a provider or the local backend is configured/running.

<a id="troubleshooting"></a>
## Troubleshooting

- Run `make litellm-status` and `make litellm-logs`.
- If preflight fails, run `make litellm-bootstrap` and verify LiteLLM API/UI credentials and the htpasswd file were created.
- If hostname resolution fails, add `llm` and/or `llm-admin` (or your overrides) to `ENDPOINTS` and re-run hosts/DNS tooling.
- For standalone mode with remote `step-ca`, set `STEP_CA_CA_SERVER` to the remote ACME endpoint and ensure client machines trust that CA.
